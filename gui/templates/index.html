About these training parameters
Algorithm. A RandomForest regressor (ensemble of decision trees) is trained on the selected gk_input features and predicts the chosen gk_run target. Uncertainty is estimated from the tree-to-tree spread (standard deviation).
  • Test size: Fraction of rows held out for validation. 0 trains on all data and skips validation metrics.
  • Trees: Number of trees in the forest. More trees usually improve stability but take longer.
  • Max depth: Maximum depth of each tree. Blank/None allows fully grown trees.
  • Min samples leaf: Minimum rows required in a leaf node. Higher values smooth the model.
  • log1p(target): Train on log(1 + target) to reduce skew. Predictions are back-transformed; requires non-negative targets.

job_execute_large.sh Algorithm
Inputs
  - db path (batch_database_*.db)
  - optional NODES
  - flags: --analyze-only, --force-running-torun

Initialize
  - ensure gk_run has t_max_initial and t_max
  - analyze SUCCESS rows with gx_analyze.py (mark CONVERGED/RESTART)

Main loop
  - pick first TORUN row
  - if none: pick RESTART row, set restart=true in input_content, mark TORUN
  - if none: exit

Prepare run
  - if t_max_initial == 0, parse t_max from input_content
  - increment t_max by t_max_initial
  - update input_content and write input_id*_gkinputid*.in
  - set status RUNNING, record job_id, t_max_initial, t_max

Execute
  - if analyze-only: skip srun (treat as success)
  - else run GX with srun
  - on failure: status CRASHED
  - on success: status SUCCESS, run gx_analyze.py to set CONVERGED/RESTART
