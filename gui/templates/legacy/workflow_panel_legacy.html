Remote GK Inputs Workflow
Step 1: Build remote bundle (local)
  - db_update/build_remote_gk_inputs.py
  - creates remote_gk_inputs_TIMESTAMP.db + templates/

Step 2: Send bundle to Flux
  - rsync remote_gk_inputs_TIMESTAMP/ -> /u/jdominsk/DTwin/remote_gk_inputs/

Step 3: Generate inputs on Flux
  - run_remote_gk_inputs_flux.py
  - writes .in files in /u/jdominsk/DTwin/tmp_inputs
  - updates remote DB status/content

Step 4: Sync back to main DB
  - sync_remote_gk_inputs_from_flux.py
  - inserts/updates gk_input rows

  | -> gk_run.status = RUNNING -> SUCCESS/CRASHED
  | -> gk_run.job_id set per run
job_execute_large.sh Algorithm
Inputs
  - db path (batch_database_*.db)
  - optional NODES
  - flags: --analyze-only, --force-running-torun

Initialize
  - ensure gk_run has t_max_initial and t_max
  - analyze SUCCESS rows with gx_analyze.py (mark CONVERGED/RESTART)

Main loop
  - pick first TORUN row
  - if none: pick RESTART row, set restart=true in input_content, mark TORUN
  - if none: exit

Prepare run
  - if t_max_initial == 0, parse t_max from input_content
  - increment t_max by t_max_initial
  - update input_content and write input_id*_gkinputid*.in
  - set status RUNNING, record job_id, t_max_initial, t_max

Execute
  - if analyze-only: skip srun (treat as success)
  - else run GX with srun
  - on failure: status CRASHED
  - on success: status SUCCESS, run gx_analyze.py to set CONVERGED/RESTART
